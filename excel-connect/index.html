Features

Every Revit data type.
One Excel round-trip.

xlConnect gives you a consistent export → edit → import workflow for four key data types — with colour-coded Excel workbooks that any team member can read and modify.

🛏️

Room Data Export & Import

Export room names, numbers, areas, levels, heights, and finish parameters to a colour-coded workbook. Openings, Components, and per-level breakdowns each get their own sheet. Import back with an element ID–matched preview.

Rooms Openings Components
🎨

View Template Export & Import

Export every View Template to a 6-sheet workbook — basic properties, model category overrides, annotation overrides, filters, and template control. Dropdown validation for line weights, patterns, and detail levels built in.

6 sheets Dropdown validation Pattern names
✏️

Object Styles Export & Import

Export all model and annotation category Object Styles — line weights, colours, and line patterns — to a single workbook. Edit in Excel and import back to standardise styles across the entire project in seconds.

Model categories Annotation categories
🔖

Parameter Export & Import

Export element parameters from the active view to Excel. Edit values in bulk — renaming, updating notes, setting codes — then import back. Changes are shown in a diff preview before any edits are applied.

Active view elements Bulk edit
🔍

Live Diff Preview

Every import command shows a diff table before touching the model: current value, new value, and status for each row. Read-only fields are shown greyed out. Apply only when you're satisfied — or cancel and adjust the Excel file.

🗂️

Smart Template Mapping

When importing a View Template, xlConnect reads the template name and view type directly from the Excel file and auto-selects the best matching template in your project. Floor plan templates map to floor plans, sections to sections — change the mapping if needed.

🔒

Protected, Readable Workbooks

Exported workbooks are colour-coded: light blue for dropdowns, yellow for colour inputs, grey for read-only fields. Read-only cells are locked in Excel so they can't be accidentally edited. The legend is on the first sheet.

🔄

Backward Compatible

xlConnect automatically detects whether an Excel file was exported by the current version or an earlier one and applies the correct parsing logic — so old workbooks continue to import without any manual adjustments.

🧩

Works Alongside Dynamo

xlConnect uses EPPlus for Excel — no dependency on the DocumentFormat.OpenXml library that conflicts with DynamoForRevit. Load Dynamo and xlConnect in the same Revit session without crashes.

Workflow

Export. Edit. Preview. Import.

The same four-step pattern for every data type — consistent, predictable, and reversible.

01

Export from Revit

Click Export to Excel from the xlConnect ribbon tab. Choose the data type — Rooms, View Templates, Object Styles, or Parameters. A colour-coded workbook is saved to disk.

→
02

Edit in Excel

Open the workbook in Excel. Edit the cells with light blue or yellow backgrounds — these are the editable fields. Grey cells are read-only and locked. Dropdowns guide valid entries.

→
03

Preview the diff

Click Import from Excel. xlConnect reads your edited file, compares every value against the current model state, and shows a diff table — what will change, what is unchanged, what is read-only.

→
04

Apply to Revit

Review the preview and click Apply Changes. All edits are committed in a single Revit transaction — fully undoable. A summary shows exactly how many properties were updated.

  • Parameters command operates on elements in the active view
